Informe Fase I - IBEX Carwash

Major release full stack para gestión operativa de jornadas, tareas, estudiantes, clientes, vehículos, slots y reservas.

React + Vite Node.js + Express MongoDB Atlas Socket.IO AWS Amplify AWS Lightsail Cloudflare DNS

1. Introducción

IBEX Carwash Fase I es una aplicación web full stack creada para administrar la operación de un programa prelaboral. Esta versión amplía la entrega inicial incorporando catálogos relacionados, persistencia en base de datos, edición de registros, despliegue en la nube y eventos en tiempo real.

2. Objetivo

Desarrollar una major release funcional que permita gestionar clientes, tutores, estudiantes, vehículos, jornadas, tareas, slots y reservas mediante una interfaz gráfica conectada a un back-end y a una base de datos.

4. Arquitectura

La arquitectura separa responsabilidades entre front-end, back-end, base de datos, DNS y despliegue cloud. El front-end se publicó en AWS Amplify. El back-end se ejecuta en AWS Lightsail con Nginx como proxy inverso. MongoDB Atlas almacena la información persistente y Cloudflare gestiona DNS/HTTPS.

  • React + Vite: interfaz de usuario.
  • Fetch API: consumo de endpoints REST.
  • Node.js + Express: API del sistema.
  • Socket.IO: eventos en tiempo real.
  • MongoDB Atlas: persistencia.
  • AWS Amplify: hosting del front-end.
  • AWS Lightsail: servidor del back-end.
  • Cloudflare: DNS y capa de HTTPS.

7. Reglas de negocio

Los slots solo permiten horarios en intervalos de media hora y duración de 30 o 60 minutos. Las reservas se crean seleccionando cliente, vehículo y slot desde catálogos existentes, evitando inconsistencias.

9. Retos y soluciones

Uno de los principales retos fue conectar un front-end publicado en Amplify con un back-end separado en Lightsail. Se resolvió configurando variables de entorno, CORS y Nginx como proxy inverso.

Otro reto fue la persistencia. La aplicación pasó de datos temporales a MongoDB Atlas, permitiendo conservar registros después de reiniciar el servidor.

10. Conclusión

La Fase I entrega una aplicación full stack funcional, desplegada en nube, con base de datos persistente, eventos en tiempo real, interfaz responsive y operaciones CRUD completas para las entidades principales del sistema.
